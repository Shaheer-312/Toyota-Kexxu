source .venv/bin/activate


lsusb

Identify the Device Node
v4l2-ctl --list-devices

Extract Video Specs (Resolutions & Frame Rates)
v4l2-ctl -d /dev/video0 --list-formats-ext

Extract "Everything" via V4L2
v4l2-ctl -d /dev/video0 --all

Available hardware controls (Brightness, Contrast, Focus, Exposure ranges)
v4l2-ctl -d /dev/video0 --list-ctrls

Extract Hardware & Descriptor Details
Find your camera's ID: lsusb (e.g., 1234:abcd).
sudo lsusb -v -d 1234:abcd

Query System & Kernel Attributes
If you need the specific serial number or UUID for a udev rule or script
udevadm info --query=all --name=/dev/video0





sudo nano /etc/udev/rules.d/99-kexxu-cameras.rules

# Eye Camera (Microdia/Sonix) - Only the Capture Node
SUBSYSTEM=="video4linux", ATTRS{idVendor}=="0c45", ATTRS{idProduct}=="6366", ATTRS{serial}=="SN0001", ATTR{index}=="0", SYMLINK+="eye"

# Front Camera (Realtek) - Only the Capture Node
SUBSYSTEM=="video4linux", ATTRS{idVendor}=="0bda", ATTRS{idProduct}=="5842", ATTRS{serial}=="200901010001", ATTR{index}=="0", SYMLINK+="front"

Save and exit (Ctrl+O, Enter, Ctrl+X).

Unplug both cameras.

Run: sudo udevadm control --reload-rules && sudo udevadm trigger

Plug them back in.


Now we can use:
v4l2-ctl -d /dev/front --list-formats-ext
v4l2-ctl -d /dev/eye --list-formats-ext
